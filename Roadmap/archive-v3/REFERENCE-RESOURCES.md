# Reference Resources

> Resource categories for the redesigned roadmap. Prefer free, local-first, hands-on, and no-KYC resources where possible.

---

## 1. Resource Selection Criteria

Add a resource only if it is:

- [ ] Free for core learning
- [ ] Hands-on or strongly mechanism-focused
- [ ] Locally runnable when practical
- [ ] Maintained or still technically relevant
- [ ] Clear about prerequisites
- [ ] Useful for attack, defense, detection, or architecture

---

## 2. Foundations

- Linux: OverTheWire Bandit, Linux man pages, distribution documentation
- Networking: Wireshark docs, Practical packet captures, RFCs where useful
- Cryptography: CryptoPals, CryptoHack, OpenSSL docs, real TLS certificate inspection
- Threat modeling: OWASP threat modeling material, Microsoft threat modeling guidance

- OWASP Top 10
- OWASP API Security Top 10

- OWASP WebGoat
- OWASP Juice Shop
- crAPI

- MITRE ATT&CK for detection mapping

---

## Priority Rule

When overwhelmed, prioritize resources that produce artifacts:

1. A working lab
2. A tool
3. An exploit/fix pair
4. A detection rule
5. A threat model
6. An architecture decision
7. A capstone report

Reading without artifact output is supporting work, not completion.
